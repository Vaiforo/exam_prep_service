# Сервис подготовки к экзамену по численным методам

Готовый Docker-сервис для подготовки к экзамену на основе предоставленных HTML-файлов с вопросами. В проект уже встроен извлечённый банк `questions.json`: 1400 основных вопросов + 20 вопросов образца.

## Возможности

- SQLite + SQLAlchemy ORM.
- Автоматический импорт вопросов при первом запуске.
- REST API на FastAPI.
- Адаптивная генерация тестов по статистике пользователя.
- Экзамен на 20 вопросов.
- Подготовка по темам.
- Подготовка по уровню готовности: 30%, 50%, 70%, 90%, 100%.
- Подготовка по сложности: лёгкие, средние, сложные.
- Повторение ошибок.
- Хранение истории прохождений, правильных/неправильных ответов, прогресса по темам и сложности.
- Экспорт/импорт прогресса в JSON для переноса на другую машину.
- После ответа показываются правильный ответ, объяснение, теория и готовый промпт для нейросети.
- Фронтенд без Node-сборки: обычный HTML/CSS/JS, отдаётся тем же FastAPI-сервисом.

## Запуск одной командой

```bash
docker compose up --build
```

После запуска открыть:

```text
http://localhost:8000
```

API-документация:

```text
http://localhost:8000/docs
```

## Данные

При первом запуске сервис создаёт SQLite-базу и импортирует данные из:

```text
app/app_data/questions.json
```

База хранится в Docker volume `exam_prep_sqlite`. Чтобы пересоздать базу с нуля:

```bash
docker compose down -v
docker compose up --build
```

## REST API

Основные endpoints:

| Метод | URL | Назначение |
|---|---|---|
| GET | `/api/health` | проверка сервиса |
| GET | `/api/topics` | список тем |
| GET | `/api/questions` | просмотр вопросов |
| POST | `/api/tests/start` | создать тестовую сессию |
| GET | `/api/sessions/{id}` | получить сессию |
| POST | `/api/sessions/{id}/answer` | ответить на вопрос |
| POST | `/api/sessions/{id}/finish` | завершить сессию |
| GET | `/api/stats` | статистика пользователя |
| GET | `/api/errors` | список ошибок |
| GET | `/api/export` | экспорт прогресса |
| POST | `/api/import/progress` | импорт прогресса |
| POST | `/api/import/questions` | импорт нового JSON-банка вопросов |

## Пример создания экзамена

```bash
curl -X POST http://localhost:8000/api/tests/start \
  -H "Content-Type: application/json" \
  -d '{"mode":"exam","count":20}'
```

## Пример теста по уровню готовности

```bash
curl -X POST http://localhost:8000/api/tests/start \
  -H "Content-Type: application/json" \
  -d '{"mode":"readiness","readiness_level":70,"count":20}'
```

## Импорт новых вопросов

Новый банк должен быть в формате `questions.json`, совместимом с текущей структурой:

```bash
curl -X POST "http://localhost:8000/api/import/questions?force=true" \
  -F "file=@questions.json"
```

## Расширяемость

Архитектура разделена так, чтобы можно было:

- добавить новые экзамены через JSON;
- заменить SQLite на PostgreSQL через `DATABASE_URL` без переписывания моделей;
- добавить авторизацию и несколько пользователей;
- подключить LLM для генерации расширенных объяснений;
- хранить векторные эмбеддинги вопросов и делать умный подбор похожих задач.

Для PostgreSQL достаточно заменить переменную окружения, например:

```text
DATABASE_URL=postgresql+psycopg://user:password@postgres:5432/exam_prep
```

И добавить соответствующий драйвер в `requirements.txt`.

## Замечание по качеству вопросов

В `questions.json` применены базовые исправления, найденные при аудите исходных HTML-файлов: исправлены неоднозначные варианты, часть формул и пояснений, в том числе коэффициент погрешности составной формулы Симпсона для обычного шага `h`.

## Примечания к версии v14

В этой версии дополнительно перепроверены старт на чистой SQLite-базе, авторизация, пользовательские тесты, импорт/экспорт прогресса и админ-панель.

Исправлено:

- удалён дублирующий индекс SQLite, из-за которого сервис мог падать при первом старте на чистой базе;
- импорт прогресса теперь работает как полное восстановление состояния пользователя: текущие сессии, ответы и ошибки пользователя предварительно очищаются, затем загружается состояние из JSON-файла;
- исправлен текст подтверждения импорта прогресса в интерфейсе;
- проверено, что `/docs`, `/redoc` и `/openapi.json` не доступны;
- проверено, что обычный пользовательский токен не подходит для `/api/admin/*`.

Для боевого сервера обязательно задайте `.env` с `ADMIN_USERNAME`, `ADMIN_PASSWORD` и ограниченным `CORS_ORIGINS`.

## Примечания к версии v75

Изменён режим «Экзамен»: теперь он имитирует реальный экзамен по распределению сложности. По умолчанию на 20 вопросов выбирается примерно:

- 2 самых простых вопроса — 10%;
- 6 простых вопросов — 30%;
- 10 средних вопросов — 50%;
- 2 сложных вопроса — 10%.

Для другого количества вопросов пропорции сохраняются приблизительно, а итоговое количество всегда совпадает с запрошенным размером теста.
